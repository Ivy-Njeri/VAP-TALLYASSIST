Error Handling in Tally – Complete Guide

Topic 1. Introduction to Error Handling in Tally

Error handling in Tally involves identifying and fixing problems that arise during operations. These issues can come from incorrect data entry, setup mistakes, corrupted files, or software bugs. Tally provides tools and messages to help users detect and correct these errors quickly and accurately.

Topic 2. Types of Errors in Tally

  • Data Entry Errors: Mistakes during voucher or ledger entry.
  • Configuration Errors: Issues from wrong feature setup like GST.
  • Synchronization Errors: Errors in data transfer between locations.
  • Runtime Errors: Errors during operations or calculations.
  • Tally Internal Errors: Software bugs or memory problems.
  • File Corruption Errors: Damaged files due to power loss, virus, etc.

Topic 3. Common Error Messages & What They Mean

  • File Damaged! – Data is corrupted. Use repair or restore backup.
  • Memory Access Violation – Restart Tally or reinstall.
  • Invalid Voucher Type – Check and correct voucher type.
  • Tax Calculation Error – Fix incorrect tax rate or configuration.
  • Invalid Number of Arguments – TDL/report error. Remove the faulty TDL file.

Topic 4. Causes of Errors in Tally

  • Incorrect ledger configuration
  • Duplicate or missing entries
  • Corrupted data files
  • Power failure during operation
  • Wrong feature activation (e.g., GST without ledgers)
  • Manual deletion of internal files
  • Network problems during sync
  • Outdated Tally version

Topic 5. How to Identify and Fix Errors

  1. Read the error message carefully.
  2. Note down the error or code.
  3. Go to Gateway of Tally > Display > Statutory Reports > Exceptions.
  4. Use F12: Configure to filter specific errors.
  5. Fix the problem (e.g., correct a ledger, add missing data).
  6. Recheck and backup your data.

Topic 6. Using Tally’s Error Log

Tally generates an error log file automatically.

Location: C:\Users\YourName\TallyPrime\Tally.log

Steps:

  • Open the file in Notepad
  • Search for terms like "Error", "Memory", or "Corrupt"
  • Use info to fix issue or send to support

Topic 7. Troubleshooting Tips

  • Take daily backups
  • Don't shut down while Tally is running
  • Keep Tally updated
  • Use Tally’s repair tools
  • Avoid editing data files directly
  • Clear temporary files often
  • Use the Audit feature to check entries

Topic 8. Best Practices to Prevent Errors

  • Train users properly
  • Use user roles to restrict access
  • Run monthly data checks
  • Update antivirus and system software
  • Turn on Auto Backup in Tally
  • Use UPS to avoid file corruption

Topic 9. When to Contact Tally Support

Contact support if:

  • Errors are too technical
  • File repair doesn’t work
  • Data is lost or corrupted
  • System crashes during operations

Provide error messages, Tally version, and what you were doing when it happened.

Topic 10. FAQs

Q: What if I deleted a voucher?
A: Use Day Book to re-enter it.

Q: How to restore a backup?
A: Go to Company Info > Restore and select backup location.

Q: Can Tally fix corrupted files?
A: Yes, use Repair option or call support.

Q: What is a TDL error?
A: Custom file issue. Fix or remove the TDL file.
